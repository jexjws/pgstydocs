**Pigsty treats Infra & Database as Code.** You can describe everything with declarative config [**Inventory**](/config/inventory).
Which is usually an [**Ansible Inventory**](https://docs.ansible.com/ansible/latest/inventory_guide/intro_inventory.html) in YAML format: `pigsty.yml`.
But [CMDB](/config/cmdb) can also be used as dynamic inventory.

The [`configure`](/config/configure) procedure will generate a config according to your environment and input.
But it's **OPTIONAL**: you can always edit the `pigsty.yml` file directly as [**tutorial**](/config/tutorial) shows.
And there are plenty of [**templates**](/config/template) for your reference.
