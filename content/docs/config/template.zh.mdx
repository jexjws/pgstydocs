这个目录 [`conf`](https://github.com/pgsty/pigsty/blob/main/conf) 包含 pigsty 配置模板，将在 [`configure`](/zh/config/configure) 过程中使用。

配置模板可以使用 `./configure -c <conf>` 指定，其中 conf 是到 `conf` 目录的相对路径（有或没有 `.yml` 后缀）。
例如 `~/pigsty/conf/rich.yml` 可以指定为 `rich`

```bash
./configure                     # 默认使用 meta.yml 配置模板
./configure -c meta             # 明确使用 meta.yml 1 节点模板
./configure -c rich             # 使用包含所有扩展和 minio 的 1 节点模板
./configure -c slim             # 使用最小的 1 节点模板
./configure -c app/supa         # 使用 supabase 1 节点模板
```

如果没有给出 `-c|--conf`，默认会自动选择单节点 `meta` 配置模板。

这里是单节点模板，提供不同的功能和配置。

| 模板                                                              | 节点 | 描述                                                  |
|:----------------------------------------------------------------------|:----:|--------------------------------------------------------------|
| [`meta.yml`](https://github.com/pgsty/pigsty/blob/main/conf/meta.yml) |  1   | 默认 1 节点配置，pgsql、infra、node、etcd，最小扩展 |
| [`rich.yml`](https://github.com/pgsty/pigsty/blob/main/conf/rich.yml) |  1   | meta + minio + 所有扩展                               |
| [`slim.yml`](https://github.com/pgsty/pigsty/blob/main/conf/slim.yml) |  1   | meta - infra - node 监控，最小安装                 |
| [`pg18.yml`](https://github.com/pgsty/pigsty/blob/main/conf/pg18.yml) |  1   | PostgreSQL 18 集群（测试版）                                 |
