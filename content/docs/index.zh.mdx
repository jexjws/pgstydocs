## 安装 [#installation]

[**快速开始**](/zh/install/start)：[准备](/zh/prepare) 一台具有 [SSH 访问权限](/zh/prepare/admin#check-accessibility) 的 [节点](/zh/prepare/hardware#node)，新安装 [Linux](/zh/prepare/linux) 系统，
使用具有免密 [`ssh`](/zh/prepare/admin#ssh) 和 [`sudo`](/zh/prepare/admin#sudo-privileges) 权限的 [用户](/zh/prepare/admin#user) 运行：

```bash title="install"
curl -fsSL https://repo.pigsty.io/get | bash; cd ~/pigsty;  # 下载安装 Pigsty
./configure    # 生成配置文件
./install.yml  # 执行安装部署
```

[**下载**](/zh/prepare/software#pigsty)、[**配置**](/zh/config/configure) 和 [**安装**](/zh/install)。Pigsty 会在几分钟内 [**完成安装**](/zh/install)！您可以稍后 [**添加更多节点**](/zh/install/multinode) 与数据库集群。

接下来，您可以探索 [**用户界面**](/zh/install/interface)，访问 `5432` 端口上的 [Postgres](/zh/pgsql) [服务](/zh/pgsql/service)，以及 `3000` 端口上的 Grafana 监控大盘（用户名/密码：`admin` / `pigsty`）。

## 常见问题 [#faq]

<Accordions type="single" orientation={"horizontal"}>
    <Accordion title="什么是 Pigsty？">
        Pigsty 是一个 PostgreSQL 发行版——一个开源的、本地优先的托管数据库服务替代方案，如 AWS RDS。它本身不是数据库，而是一个全面的管理解决方案和经过实战检验的最佳实践集合。这样理解：如果数据库是汽车，DBA 是司机，那么 RDS 就像优步，而 Pigsty 是您的个人自动驾驶系统。
    </Accordion>

    <Accordion title="Pigsty 解决什么问题？">
        数据库专业知识昂贵且难以找到——您要么雇佣昂贵的 DBA，要么为云 RDS 支付高昂价格。Pigsty 提供第三种选择：在自己的基础设施中运行企业级数据库，无需专门的 DBA，成本不到典型 RDS 的 10%，同时获得更好的性能和灵活性。
    </Accordion>

    <Accordion title="为什么选择 Pigsty 而不是 RDS？">
        Pigsty 提供大多数 RDS 提供商无法匹敌的功能：<span className={"text-red-800 font-bold"}>420+</span> PostgreSQL 扩展，支持 8+ Postgres 变体，世界级监控，以及经过苹果、阿里巴巴等公司大规模验证的实战架构。它带着真正的热情构建，达到了典型 RDS 平台根本无法提供的功能深度。
    </Accordion>

    <Accordion title="使用 Pigsty 能节省多少成本？">
        显著节省。在 RDS 上每 vCPU 每月花费 $100-200 的成本，使用 Pigsty 在裸机上运行只需每 vCPU 每月 $2——还省下了 DBA 工资。在规模化时，总拥有成本可降低 90% 或更多，释放预算用于对业务真正重要的事情。
    </Accordion>

    <Accordion title="Pigsty 如何帮助开发者？">
        凭借 <span className={"text-red-800 font-bold"}>420+</span> 内置 PostgreSQL 扩展，Pigsty 实现了 "一切皆用 Postgres" 的方法。用单一、统一的技术栈替换 Redis、Kafka、MySQL、Elasticsearch、向量数据库，甚至一些 OLAP 工作负载。这减少了复杂性，提高了开发者速度，实现了真正的自服务 DevOps。
    </Accordion>

    <Accordion title="运维和可靠性如何？">
        Pigsty 的自愈高可用意味着硬件故障时不再有凌晨 3 点的叫醒电话。高级监控有助于故障排除和性能优化。基础设施即代码自动化大规模集群管理。您的运维团队可以处理 DBA 任务，您的 DBA 可以专注于高价值工作而不是救火。
    </Accordion>

    <Accordion title="谁在开发 Pigsty？">
        Pigsty 主要由冯若航（@Vonng）开发，他是拥有十多年开源贡献和在阿里巴巴、探探、苹果生产经验的 PostgreSQL 资深专家。他也是微信上拥有 6 万多粉丝的顶级数据库意见领袖，现在经营一家专注于 PostgreSQL 卓越的咨询公司。
    </Accordion>

    <Accordion title="Pigsty 支持什么规模？">
        Pigsty 可扩展从单个开发者设置到大规模企业部署。已知最大的生产部署在六年内处理 2.5 万 vCPU、450 万 QPS。最小的？一个简单的 1 核演示虚拟机。它为企业规模而设计，但为简单性而优化。
    </Accordion>

    <Accordion title="Pigsty 真的免费吗？">
        是的，Pigsty 是 AGPLv3 许可下的 100% 自由软件。只要遵守许可证条款，您可以商业使用。我们相信软件自由——对于大多数用例，我们采用类似 Apache 2.0 的方法。对于想要专业支持和额外许可选项的用户，可以购买商业订阅。
    </Accordion>

    <Accordion title="我可以用 Pigsty 构建 DBaaS 吗？">
        可以，在 AGPLv3 许可下——这意味着任何衍生作品都必须开源。我们积极防范云提供商的许可违规。如果您不想开源衍生作品，Pigsty 企业订阅为这种场景提供明确的许可和 AGPLv3 豁免。
    </Accordion>
</Accordions>
