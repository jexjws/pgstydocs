这里有一个检查清单，帮助您为生产环境中的严肃 Pigsty 部署准备环境。

|               项目                | 要求                                           |                       项目                       | 要求                                                     |
|:---------------------------------:|:----------------------------------------------|:------------------------------------------------:|:--------------------------------------------------------|
|  [节点](/zh/prepare/hardware#node)   | 至少 `1C1G`，推荐 `2C2G`，无上限                |          [规格](/zh/prepare/hardware#spec)          | 至少 `1` 个节点，`2` 个用于半高可用，`3+` 个用于真正的高可用        |
|  [磁盘](/zh/prepare/hardware#disk)   | `/data`，主挂载点，`ext4` 或 `xfs`               |       [网络](/zh/prepare/hardware#network)       | 静态内网，IPv4 地址，最好有互联网访问                          |
|   [VIP](/zh/prepare/hardware#vip)    | 为 VIP 保留一个 L2 IP （**可选**）                |        [域名](/zh/prepare/hardware#domain)        | 使用本地 / 公共域名（**可选**）                            |
| [内核](/zh/prepare/software#linux) | `Linux`，MacOS 可用作管理控制器                  |             [发行版](/zh/prepare/linux)             | EL (8/9)、Debian (12)、Ubuntu (22/24)、`x86_64` / `aarch64` |
| [区域设置](/zh/prepare/software#linux) | `C.UTF-8` 或 `C`                            |      [防火墙](/zh/prepare/software#firewall)      | 端口：`80` / `443` / `22` / `5432`                      |
|    [用户](/zh/prepare/admin#user)    | 避免使用 `root` 和 `postgres`                  |      [Sudo](/zh/prepare/admin#sudo-privileges)      | `nopass` sudo 权限                                     |
|     [SSH](/zh/prepare/admin#ssh)     | 通过公钥 `nopass`                             | [可访问](/zh/prepare/admin#check-accessibility) | `ssh <ip\|alias> sudo ls` 有效                         |
